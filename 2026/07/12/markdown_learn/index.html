二、字体样式

星号 * 与下划线 _ 效果相同，前后一个为斜体、两个为粗体、三个为粗斜体。

语法 效果
*斜体* 或 _斜体_ 斜体
**粗体** 或 __粗体__ 粗体
***粗斜体*** 或 ___粗斜体___ 粗斜体

三、换行与分隔

3.1 换行

  1. 直接回车：在部分解析器中有效（如 GitHub），但建议显式换行。
  2. <br/>：HTML 标签，兼容性最好。

3.2 分隔线

三个及以上的 *、- 或 _，行内不能有其他字符（空格除外）。

九、列表

9.1 无序列表

使用 *、+ 或 - 外加一个空格：

  • 项目一
  • 项目二
    • 子项目 A
    • 子项目 B

9.2 有序列表

使用 数字. + 空格：

  1. 第一步
  2. 第二步
  3. 第三步

💡 提示：数字不必连续，渲染时会自动排序。

十、表格

用 | 分隔单元格，用 - 分隔表头与内容行，并用 : 控制对齐方式：

对齐方式 语法 说明
左对齐 :- 内容靠左
右对齐 -: 内容靠右
居中对齐 :-: 内容居中

示例：

项目 价格 数量
计算机 $1600 5
手机 $12 12
管线 $1 234

十一、空行的作用

  1. 分隔段落：让不同段落之间保持清晰间距。
  2. 分隔元素：在标题、列表、代码块等不同元素之间留出空白，提升可读性。

✅ 最佳实践：不同语义块之间保留一个空行，是 Markdown 排版的黄金法则。
